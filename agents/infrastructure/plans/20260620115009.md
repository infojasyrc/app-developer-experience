## Scope

Deploy `ms-conference-api` (NestJS, port 3000) and `ms-conference-webapp` (Next.js, port 3000)
on ECS Fargate, plus a **separate ECS service** running platform tools:
Unleash, Grafana, and Keycloak.

---

## Architecture Overview

```
Internet
  │
  └─ ALB (app-alb, public) ──WAF──► SG: alb-app
       ├── api.domain  → TG: api  → ECS service: cm-app (API task, port 3000)
       └── app.domain  → TG: webapp → ECS service: cm-app (Webapp task, port 3000)

VPC-internal
  └─ ALB (tools-alb, internal) ──► SG: alb-tools
       ├── unleash.internal → TG: unleash → ECS service: cm-tools (port 4242)
       ├── grafana.internal → TG: grafana  → ECS service: cm-tools (port 3000)
       └── keycloak.internal → TG: keycloak → ECS service: cm-tools (port 8080)

ECS Cluster: appdevexp-dev (shared)
  ├── Service: cm-app   (API + Webapp tasks)  ← private subnets, no public IP
  └── Service: cm-tools (Unleash + Grafana + Keycloak) ← private subnets, no public IP

DocumentDB Cluster (MongoDB-compatible): cm-docdb ← private subnets
Secrets Manager: credentials for DB, Keycloak, Unleash, Firebase
```

**Key design decisions:**
- One ECS cluster, two ECS services (application and tools are isolated by separate services, security groups, and load balancers).
- Tools ALB is **internal** (not internet-facing); only reachable within VPC or via VPN.
- The existing `module/application` is refactored into two new modules: `module/ecs-app` and `module/ecs-tools`.
- All resources carry `project = var.application_name` tag for `make list-resources`.
- DocumentDB replaces the commented-out EC2/RDS database module.
- All secrets go into Secrets Manager and are injected via ECS `secrets` block (not env vars).

## Phase 3 — Application ECS Service (API + Webapp)

> Replaces the commented-out `module/application`. Refactored into `module/ecs-app`.

### 3-A: Create `cloud/terraform/aws/module/ecs-app/`

The module creates:
- Security Group for ALB (internet-facing, HTTPS 443 only)
- Security Group for ECS tasks (ingress from ALB SG on port 3000 and 3001)
- External ALB with WAF association
- Target Groups for API (port 3000) and Webapp (port 3001)
- ALB Listener (HTTPS/443) with host-header routing rules
- Route 53 CNAME records
- Two ECS Task Definitions (api-task, webapp-task)
- One ECS Service per task (or a combined service with two tasks)
- CloudWatch Log Group (per service)

**`module/ecs-app/main.tf`** — Security Groups:

## Tagging Strategy

All Terraform-managed resources use `local.common_tags`:
```hcl
locals {
  common_tags = {
    project     = var.application_name   # "appdevexp" → used by make list-resources
    environment = local.environment       # "dev"
    managed_by  = "terraform"
    solution    = "conference-manager"
  }
}
```

Every `module` call passes `tags = local.common_tags`.  
Every inline resource (`aws_security_group`, `aws_lb`, etc.) uses `merge(var.tags, {...})`.

`make list-resources` queries `Key=project,Values=appdevexp` — this covers all resources.
