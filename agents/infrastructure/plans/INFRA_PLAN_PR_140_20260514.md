`aws s3api head-bucket --bucket tf-state-ade-bucketex-unique` → **404 Not Found**
`aws s3api list-buckets` → `tf-state-ade-bucketexp-unique` **exists**

### Cross-reference with static findings

The template `iam/terraform-backend-policy.json.tpl` is correct and uses
`__TF_STATE_BUCKET__`. The drift was introduced at bootstrap time: when
`make bootstrap-deployer` was first run, `TF_STATE_BUCKET` was set to
`tf-state-ade-bucketex-unique` (missing "p"). `update-deployer` was never
re-run to correct it.
