## Root Cause Analysis

The `TerraformBackendPolicy` managed policy that previously granted
`GitHubActionsTerraformRole` S3 state access was deleted between run 25714513898
(2026-05-12) and run 25839064759 (2026-05-14). Likely cause: admin attempted to
fix the missing `AssumeServiceRoles` SID (per prior plan) by deleting and
recreating the policy but never re-attached it.

`make update-deployer` had no effect because it hardcodes
`DEPLOYER_ROLE := appdevexp-deployer` — no Makefile target covers
`GitHubActionsTerraformRole`.

## Fix Actions (admin only — do not run via Claude)

### Option A — Apply inline backend policy to `GitHubActionsTerraformRole` (preferred)

```bash
cd cloud/terraform/aws

# Step 1 — render the template with correct values
make render-backend-policy \
  TF_STATE_BUCKET=tf-state-ade-bucketexp-unique \
  TF_LOCK_TABLE=ade-terraform-lock-table \
  AWS_ACCOUNT_ID=580976914278 \
  AWS_REGION=us-west-1 \
  GITHUB_USER=infojasyrc \
  REPO_NAME=app-developer-experience \
  APP_NAME=appdevexp

# Step 2 — apply to the role AWS_ROLE_ARN actually uses
aws iam put-role-policy \
  --role-name GitHubActionsTerraformRole \
  --policy-name terraform-backend \
  --policy-document file:///tmp/appdevexp/terraform-backend-policy.json

# Step 3 — verify
aws iam get-role-policy \
  --role-name GitHubActionsTerraformRole \
  --policy-name terraform-backend \
  --query PolicyDocument --output json
```

No secrets change. No `.tf` or workflow file modification required.

`appdevexp-deployer` is the Makefile-bootstrapped OIDC role with correct policy.
